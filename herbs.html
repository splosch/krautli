DataStorage: WebSQL Database to store data on Client (Offline mode by default)

Since the whole application should be able to run on the User phone without the immediate need to communicate back to the server, we need a way to store users data & application data on the clientside.
The Data should be persisted and datarelation is needed to reference e.g. plantnames & their synonyms, as well as their positiondata & harvesting information. I picked the WebSQL DB build in Webkit Browsers like Chrome or Safari, but like to have a look on other systems too - the "worst" fallback would be the Synchronous communication back to the Server which implements a DB on its own. Idea: What? Why? How?

This tool is a self-lerning-project to show what's possible with a non-device-spcific coding approach that shows all kind of new features implemented in the latest browsers. The Code will be fully documented and will go through various levels of refactoring and improvements.
With following this project YOU should be aware of the possibilities and limitations of the HTML5 + all the APIs going hand in hand (*sometimes).
Please don't bash us for writing bad code in a horrible style with absolute no structure - this will change within that project.
This HTML & JS Application is ment to be a helper tool for dudes & maa's that like to collect herbals and like to build up a collection of nice places where to find them.
These are the main features I'm aiming for

  1. Select Plants
    • Avoid extra typing effort (autocomplete / suggestions) (DONE)
    • Provide all needed Data for offline usage (*not yet)
    • Select Synonyms of Plantnames (*maybe)
    • provide new Plantnames (*not yet)
    • "name later" - entry, only together with notes & images / considder bulk naming of multiple entries (*not yet)
    • Fallback: server side handling of unknown plants or other processing
  2. Get the position of the Place where you found the Plant
    • Use GPS Position (DONE)
    • make sure this works offline (*not yet)
    • Fallback 1: use network based Positioning (*buggy)
    • Fallback 2: select position manually (freetext, gps, pinpoint on a map, ...) (*not yet)
    • Fallback 3: no position (e.g. for creating new plants) (*maybe)
  3. Store the Position of the Plant
    • Store everything on Clientside (*half done / buggy)
    • Allow upload of Data per Customer (*not yet)
    • Private / Public entries (*maybe)
    • make sure this works offline (*DONE)
    • Fallback 1: try LocalStorage for WebSQL fallback(*maybe)
    • Fallback 2: allow direct Save on Serverside (*maybe)
  4. Display stored Plants
    • display the last saved position (DONE + disabled)
    • display all Plants in Textform
    • display all Plants on a Map
    • display only selected Plants on a Map
    • filter Plants by (Erntezeitpunkt Pflanzenteile, etc)
    • make this offline ?? (possible?)
    • display shared Plantlocations by other users
  5. Other Functions
    • edit / delete Plants
    • approve / mark wrong shared locations
    • store notes & images along with the position
  6. Technical Requests
    • Get a functional Prototype working (somehow)
    • clean package structuring within a Git Project
    • OOP & MVC
    • Event based Interaction on Client & Server Side
    • Integrate node.js server for correspondence & fallback support
